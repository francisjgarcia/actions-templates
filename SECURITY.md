# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| `@main` | ✅ Always supported |
| Tagged releases | ✅ Latest tag supported |
| Older tags | ❌ Not supported |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

To report a vulnerability, use one of the following options:

### Option 1 — GitHub Security Advisories (preferred)

Open a private advisory at:
👉 https://github.com/francisjgarcia/actions-templates/security/advisories/new

### Option 2 — Email

Send a detailed report to **contacto@francisjgarcia.es** with the subject:
`[SECURITY] actions-templates - <short description>`

Please include:

- Description of the vulnerability
- Affected workflow or action
- Steps to reproduce
- Potential impact
- Suggested mitigation (if any)

## Response time

| Stage | Target time |
| --- | --- |
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix or mitigation | Depends on severity |

## Disclosure policy

Once a fix is available, the vulnerability will be disclosed publicly via a
GitHub Security Advisory. Credit will be given to the reporter unless they
prefer to remain anonymous.
